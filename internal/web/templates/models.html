Quick Connect

This proxy exposes a standard OpenAI-compatible API at http://{{.DisplayAddr}}/v1. Point any OpenAI-compatible client at that URL with your proxy API key.

Connect Zed Editor

Zed uses the OpenAI provider for custom endpoints. Add the following to your ~/.config/zed/settings.json:

Step 1 — Add to settings.json
~/.config/zed/settings.json 
{
  "language_models": {
    "openai": {
      "api_url": "http://{{.DisplayAddr}}/v1",
      "available_models": [
        {{- range $i, $b := .Backends}}{{if $b.Enabled}}{{range $j, $m := $b.Models}}{{if or $i $j}},
        {{end}}{
          "name": "{{$m}}",
          "display_name": "{{$m}} (proxy)",
          "max_tokens": 8192
        }{{end}}{{end}}{{end}}
      ]
    }
  },
  "assistant": {
    "default_model": {
      "provider": "openai",
      "model": "{{.SampleModel}}"
    },
    "version": "2"
  }
}
Step 2 — Set your API key

Zed reads the OpenAI key from OPENAI_API_KEY. Set it in your shell profile:

~/.zshrc or ~/.bashrc 
export OPENAI_API_KEY="your-proxy-api-key"

Reload Zed after saving. The models will appear in the assistant panel under the OpenAI provider.

Connect VS Code Continue

Open the Continue extension settings (Ctrl+Shift+P → Continue: Open config) and add the following models to your ~/.continue/config.yaml:

~/.continue/config.yaml 
models:
{{- range $i, $b := .Backends}}{{if $b.Enabled}}{{range $b.Models}}
  - name: "{{.}} (proxy)"
    provider: openai
    model: "{{.}}"
    apiBase: "http://{{$.DisplayAddr}}/v1"
    apiKey: "your-proxy-api-key"
{{end}}{{end}}{{end}}

Replace your-proxy-api-key with one of the keys from your proxy config.yaml. Save the file — Continue reloads automatically.

curl / HTTP Examples

List all models
bash 
curl http://{{.DisplayAddr}}/v1/models \
  -H "Authorization: Bearer your-proxy-api-key"
Chat completion (non-streaming)
bash 
curl http://{{.DisplayAddr}}/v1/chat/completions \
  -H "Authorization: Bearer your-proxy-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "{{.SampleModel}}",
    "messages": [{"role": "user", "content": "Hello!"}]
  }'
Streaming chat completion
bash 
curl http://{{.DisplayAddr}}/v1/chat/completions \
  -H "Authorization: Bearer your-proxy-api-key" \
  -H "Content-Type: application/json" \
  --no-buffer \
  -d '{
    "model": "{{.SampleModel}}",
    "messages": [{"role": "user", "content": "Count to 5"}],
    "stream": true
  }'
Health check
bash 
curl http://{{.DisplayAddr}}/health
