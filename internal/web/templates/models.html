    Connect Zed Editor

    Zed uses the OpenAI-compatible provider for custom endpoints. Add the following to your ~/.config/zed/settings.json:

    Step 1 — Choose flattened models
    ~/.config/zed/settings.json 
    Loading models…
    Step 2 — Set your API key

    Zed reads the key from the environment variable LLM_API_PROXY_API_KEY. Set it in your shell profile:

    ~/.zshrc or ~/.bashrc 
    export LLM_API_PROXY_API_KEY="your-proxy-api-key"

    Reload Zed after saving. The generated config uses flattened model IDs, so the proxy decides which backend should serve each request.

    Connect OpenCode

    OpenCode can use this proxy as an OpenAI-compatible provider. Add the following to your ~/.config/opencode/opencode.json:

    Step 1 — Choose flattened models
    ~/.config/opencode/opencode.json 
    Loading models…
    Step 2 — Set your API key

    OpenCode should read the key from the environment variable LLM_API_PROXY_API_KEY. Set it in your shell profile:

    ~/.zshrc or ~/.bashrc 
    export LLM_API_PROXY_API_KEY="your-proxy-api-key"
    Step 3 — Reload OpenCode

    Restart OpenCode after saving. The generated config points at a single proxy provider and uses flattened model IDs so routing stays inside the proxy.

    Connect Droid

    Droid/Factory can load custom OpenAI-compatible models. Add the following to your ~/.factory/settings.json:

    Step 1 — Choose flattened models
    ~/.factory/settings.json 
    Loading models…
    Step 2 — Set your API key

    Droid supports environment variable interpolation in settings.json. Set LLM_API_PROXY_API_KEY in your shell profile:

    ~/.zshrc or ~/.bashrc 
    export LLM_API_PROXY_API_KEY="your-proxy-api-key"
    Step 3 — Refresh Droid

    Restart Droid or reload its settings. The generated models are flattened bare IDs, and the proxy handles backend routing internally.

    VS Code Copilot Extension
    
                
            
    
            
    
                
    
                    A companion VS Code extension that discovers models from this proxy dynamically and surfaces them
                    directly in the GitHub Copilot Chat model picker — no hardcoding required.
                
    

                
    Step 1 — Install the extension
    
                
    
                    Install LLM API Proxy — Copilot Chat from the VS Code Marketplace:
                
    
                
    
                    
    
                        bash
                        
                    
    
                    
    code --install-extension vleeuwenmenno.llmapiproxy-vscode-chat
    
                
    
                
    
                    Or search for "LLM API Proxy" in the Extensions view (Ctrl+Shift+X).
                
    

                
    Step 2 — Connect to this proxy
    
                
    Open the Command Palette (Ctrl+Shift+P) and run LLM API Proxy: Manage Connection, then:
    
                
      
                    
    • Choose Set / Update API Key and enter your proxy API key
      • 
                    
    • Optionally choose Change Proxy URL if your proxy is not at http://localhost:8000
      • 
                
    

                
    Step 3 — Use it
    
                
    Open Copilot Chat (Ctrl+Alt+I), click the model selector, and pick any model prefixed with a backend name. Run LLM API Proxy: Refresh Available Models any time to re-sync.
